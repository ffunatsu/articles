GW連休後半、九州は空も快晴で心地が良い。最近雨ばっかりで憂鬱だったので、少し嬉しい。（やっと布団が干せた。）

さて、先日GitHub Copilotの利用停止について[記事を書いた](260504_GitHub_Copilotの利用停止)が、結論としてGemini Code Assistに移行することにした。

https://codeassist.google/

https://aismiley.co.jp/ai_news/gemini-code-assist/

理由としては、AIアシストを全く使わなくても普段のコーディングにはほぼ何も支障はないのだけれど、原因不明のバグに陥ったときに、やはり自分だけで探索するのはかなり時間がかかり、数時間、数日かかるようなこともある。そうしたときだけにピンポイントに使用するコーディングAIアシスタントはやはり欲しかった。

自分の使用用途では、Gemini Code Assistなら今のところ無料の範囲に収まる様子で、一応（現時点で）$19/月 の上位プランもあるらしい。$19/月くらいが自分が出せる限界かなと感じていて、自分は普段は全く生成AIを使わないし、かといってバグ等にハマってしまったときに議論が白熱すると、GitHub Copilotの無償枠には収まらない（※ 以前はかなり上限が小さかったが、従量課金制になったあとのことは実は知らない）ので、自分にはAnthropic等ではなくGemini Code Assistが今のところちょうどいいかなと思った。

Geminiは軽いAIとして以前から興味はあって、ローカルLLMとしてのGemma 4もいざというときには良いかなくらいに感じていたのだけれど、Gemma 4 は（モデルサイズの選択肢はあるけれど）自分のPCでローカルに使っている分にはちょっとクラウドAIとの議論とはさすがに比べ物にはならない使い勝手で、他のローカルLLMも同様で結局使っていないのが現状。

ちなみにGeminiは [TrussCのGetting Started](https://github.com/TrussC-org/TrussC)でTrussC Assistantが作られているので時々TrussCの質問をしていたりして、こういう使い方もできるんだなと思っていた。（GPTsなどで似たようなことはできるはずだけれど、README等からURLですぐ導入できるものは自分はあまり知らなかったので新鮮だった。）

Geminiは少なくともGPT-5やAnthropicなどの巨大AIほどのリソースを食っていることは今のところはなさそうなので、Gemini Code Assist、たまの議論の相手としてたまに使ってみて様子をみよう。

（余談として、昔はCodyが大好きで使っていた時期もあったのだけれど、Codyはサービス終了してしまったので残念。ただ、今もCodyがあったとしても、Codyを愛用していた時期くらいからAI疲れが酷くなっていたので、結局は同じように使っていなかったかもしれない。AIはアメリカや世界の電力問題を酷くしているということに危機感を持っているのもあるし。）
